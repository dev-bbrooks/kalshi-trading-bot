Visual overhaul of terminal.py's HTML/CSS to match the main dashboard app at dash.btcbotapp.com. Read terminal.py first. This is CSS/HTML only — do NOT touch any Python backend code or JavaScript logic.

## The Problem

The terminal uses hardcoded colors and a different design language than the dashboard. They need to feel like the same app. Here's exactly what to change:

## 1. CSS Variables

Replace all hardcoded colors with the dashboard's CSS variable system. Add this to the top of the <style> block:

```css
:root {
    --bg: #0d1117;
    --card: #161b22;
    --border: #30363d;
    --text: #c9d1d9;
    --green: #3fb950;
    --red: #f85149;
    --yellow: #d29922;
    --blue: #58a6ff;
    --dim: #8b949e;
    --orange: #f0883e;
}
```

Then replace every hardcoded color:
- #0a0a0a → var(--bg)
- #111, #1a1a1a → var(--card)
- #333, #2a2a2a → var(--border)
- #e0e0e0, #c9d1d9 → var(--text)
- #888, #666, #555, #aaa, #777 → var(--dim)
- #4a9eff → var(--blue)
- #22c55e → var(--green)
- #f59e0b → var(--yellow)
- #ef4444 → var(--red)
- #1a1a2e (user message bg) → rgba(88, 166, 255, 0.08) (subtle blue tint using the blue variable)

## 2. Font Family

Change from `system-ui, -apple-system, sans-serif` to match dashboard:
```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

Monospace stays: `'SF Mono', Menlo, Monaco, monospace`
